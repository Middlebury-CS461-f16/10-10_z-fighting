Z-Fighting

Here we have two triangles that are drawn at the same depth. In theory, they should now be drawn in the order that we requested them to be rendered. However, precision errors lead to z-fighting, where the interpolated pseudodepth values vary and the triangles seem to be melting into one another.

We can address this by using polygon offset, which is a hack that allows us to add an offset to the depth after the projection.
